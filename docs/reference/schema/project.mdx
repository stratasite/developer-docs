# project.yml Schema

Project configuration file schema.

## Overview

`project.yml` defines your Strata project metadata and configuration.

## Location

Root of your project directory.

## Schema

```yaml
name: string                    # Required: Display name
description: string            # Optional: Project description
uid: string                    # Optional: URL-friendly ID
server: string                # Required: Strata server URL
production_branch: string      # Optional: Production branch name (default: main)
git: string                    # Optional: Git repository URL
project_id: integer           # Auto-populated: Server project ID (do not change)

# Multi-environment support
environments:
  environment_name:
    server: string             # Environment-specific server URL
    api_key: string            # Environment-specific API key
```

## Required Fields

### name

Display name for the project.

```yaml
name: My Analytics Project
```

### server

Strata server URL.

```yaml
server: https://strata.example.com
```

## Optional Fields

### description

Project description.

```yaml
description: Analytics project for sales data
```

### uid

URL-friendly identifier.

```yaml
uid: my-analytics-project
```

### production_branch

Git branch name for production deployments.

```yaml
production_branch: main
```

**Default:** `main`

### git

Git repository URL.

```yaml
git: https://github.com/example/strata-project.git
```

### project_id

Server project ID. Auto-populated after first deployment. **Do not change manually.**

```yaml
project_id: 123
```

## Multi-Environment Configuration

Define environment-specific settings:

```yaml
environments:
  production:
    server: https://strata.example.com
    api_key: prod_key_here
  staging:
    server: https://staging.strata.example.com
    api_key: staging_key_here
```

**Usage:**
```bash
strata deploy --environment production
```

## Complete Example

```yaml
name: TPC-DS Tutorial
description: A complete model of the TPC-DS benchmark system
production_branch: main
server: http://localhost:3000
git: https://github.com/stratasite/tpcds-tutorial
project_id: 1  # WARNING: Do not change - links to Strata server
```

## Best Practices

1. **Set production_branch** - Match your Git workflow
2. **Use environments** - Separate staging and production
3. **Don't change project_id** - Auto-managed by server
4. **Add description** - Document project purpose
