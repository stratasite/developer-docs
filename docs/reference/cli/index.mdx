# CLI Reference

Complete reference for all Strata CLI commands.

## Overview

The Strata CLI provides commands for managing semantic models, datasources, and deployments.

## Command Structure

Commands follow the pattern: `strata [COMMAND] [SUBCOMMAND] [OPTIONS]`

## Available Commands

### Project Management
- **[init](/docs/reference/cli/init)** - Initialize a new Strata project
- **[project link](/docs/reference/cli/init#project-link)** - Link to existing server project

### Datasource Management
- **[datasource](/docs/reference/cli/datasource)** - Manage datasource connections
  - `add` - Add new datasource
  - `list` - List configured datasources
  - `auth` - Set credentials
  - `test` - Test connection
  - `tables` - List tables
  - `meta` - Show table schema
  - `exec` - Execute SQL

### Model Creation
- **[create](/docs/reference/cli/create)** - Create semantic models
  - `table` - Create table model
  - `relation` - Create relationship file
  - `migration rename` - Create rename migration
  - `migration swap` - Create swap migration

### Table Management
- **[table list](/docs/reference/cli/table)** - List semantic tables

### Deployment
- **[deploy](/docs/reference/cli/deploy)** - Deploy project to server
- **[deploy status](/docs/reference/cli/deploy#status)** - Check deployment status

### Audit & Validation
- **[audit](/docs/reference/cli/audit)** - Run validation checks
  - `all` - Run all checks (default)
  - `syntax` - Check YAML syntax
  - `models` - Validate models
  - `connections` - Test connections

### Utilities
- **version** - Print CLI version
- **adapters** - List supported adapters

## Getting Help

Get help for any command:

```bash
strata COMMAND --help
strata COMMAND SUBCOMMAND --help
```

## Command Aliases

- `t` / `tbl` → `table`
- `ds` → `datasource`
- `a` → `audit`
