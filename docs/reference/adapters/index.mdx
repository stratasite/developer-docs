# Adapters

Strata supports 8 database adapters for connecting to your data warehouses.

## Supported Adapters

1. **[PostgreSQL](/docs/reference/adapters/postgres)** - Full PostgreSQL support
2. **[Snowflake](/docs/reference/adapters/snowflake)** - Snowflake data warehouse
3. **[MySQL](/docs/reference/adapters/mysql)** - MySQL database
4. **[SQL Server](/docs/reference/adapters/sqlserver)** - Microsoft SQL Server (including Azure)
5. **[Athena](/docs/reference/adapters/athena)** - AWS Athena
6. **[Trino](/docs/reference/adapters/trino)** - Trino/Presto
7. **[DuckDB](/docs/reference/adapters/duckdb)** - Embedded analytics database
8. **[Druid](/docs/reference/adapters/druid)** - Apache Druid

## Common Configuration

All adapters support:

- **adapter** - Adapter name (required)
- **name** - Display name (required)
- **description** - Optional description
- **tier** - Performance tier: `hot`, `warm`, `cold` (default: `warm`)
- **query_timeout** - Query timeout in seconds (default: `3600`)
- **client_name** - Client name sent to database (default: `Strata`)

## Adding Adapters

Use the CLI to add adapters:

```bash
strata datasource add postgres
strata datasource add snowflake
```

Or edit `datasources.yml` directly.

## Authentication

Credentials are stored securely in `.strata` (not committed to git):

```bash
strata datasource auth datasource_key
```
